## 2026-05-21

**판단 실수**: `getServerSideProps`에서 `items` 배열을 명시적으로 필드 매핑했는데, `itemId`를 포함하는 코드를 작성하지 않아 클라이언트에 `itemId`가 전달되지 않았다. 결과적으로 battles/records에 `winnerItemId`가 저장되지 않았고, 여러 태스크를 완료한 후에야 발견됐다.

**교정 방향**: `getServerSideProps`에서 객체를 명시적으로 매핑할 때, 새로 추가한 필드가 해당 매핑에도 포함되어 있는지 반드시 확인한다. 새 필드 추가 후 저장→SSR 전달→클라이언트 사용 흐름을 end-to-end로 검증한다.

## 2026-05-21

**판단 실수**: `getUploadDir()`의 개발 환경 분기에서 `/tmp`를 하드코딩했다. Windows에서는 `D:\tmp`로 해석되어 해당 폴더가 없으면 ENOENT 오류가 발생했고, 로컬에서 파일 업로드가 처음부터 작동하지 않았다.

**교정 방향**: 경로를 하드코딩하지 않고 `os.tmpdir()`을 사용해 OS별 기본 임시 디렉토리를 참조한다. 로컬 개발 환경(특히 Windows)에서도 핵심 기능이 정상 동작하는지 초기에 확인한다.
